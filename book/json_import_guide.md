# الدليل الشامل لهيكلة بيانات التمارين والمواضيع (JSON)

هذا الدليل مخصص للأشخاص الذين سيقومون بكتابة أو إدخال المواضيع والتمارين لمنصة **SujetStore**. يعتمد النظام على هيكل بيانات JSON دقيق وموحّد. في هذا الدليل سنشرح كيف يتم كتابة تمرين عادي، وكيف تُكتب الوضعية الإدماجية، وكيف يتم تجميعها في موضوع (امتحان أو فرض) كامل. 

نحن نركز هنا على **السنة الأولى متوسط (1am) - مادة الرياضيات** لأي فصل دراسي، لكن القالب ينطبق على كل المواد والمستويات.

---

## 1. قالب تمرين منفرد (ExerciseBlock)

يُستخدم هذا القالب من أجل إدخال "تمرين واحد" فقط إلى بنك التمارين، لكي يستطيع الأستاذ لاحقاً ادراجه في أي موضوع. 

```json
{
  "points": 3,
  "num": "التمرين الأول",
  "instruction": "إليك الأعداد التالية:",
  "content": [
    {
      "type": "text",
      "content": "A = 15.2, B = 3.14, C = 10",
      "answer": "لا يوجد جواب هنا، المعطيات فقط",
      "mark": "0"
    },
    {
      "type": "text",
      "content": "1) رتّب هذه الأعداد ترتيباً تصاعدياً.",
      "answer": "B < C < A",
      "mark": "1.5"
    },
    {
      "type": "math",
      "content": "A + B = ?",
      "display": false
    },
    {
      "type": "text",
      "content": "2) احسب المجموع.",
      "answer": "18.34",
      "mark": "1.5"
    }
  ]
}
```

### شرح حقول التمرين:
- `points` (رقم أو نص): عدد النقاط الإجمالية للتمرين.
- `num` (نص، اختياري): ترتيب التمرين (مثال: "التمرين الأول"). إذا لم يتم إدخاله، سيقوم النظام بترقيمه آلياً.
- `instruction` (نص، اختياري): التوجيه أو التعليمة التي تأتي في بداية التمرين قبل الأسئلة.
- `content` (قائمة []): قائمة من "مكعبات المحتوى" (Content Blocks). كل سؤال، نص، أو معادلة يعتبر مكعباً.

## 2. قالب وضعية إدماجية (Integrating Situation)

الوضعية الإدماجية (بالنسبة للرياضيات 1 متوسط) هي رياضياً مجرد "تمرين" كبير يمتلك تنقيطاً أعلى (8 نقاط غالباً) وسياقاً أطول. لذلك نفس القالب يُستخدم، مع تغيير القيم لتناسب الوضعية.

```json
{
  "points": 8,
  "num": "الوضعية الإدماجية",
  "instruction": "الجزء الأول: يملك فلاح قطعة أرض مستطيلة الشكل...",
  "content": [
    {
      "type": "text",
      "content": "طول القطعة 120m وعرضها 80m."
    },
    {
      "type": "text",
      "content": "1) احسب مساحة هذه القطعة.",
      "answer": "المساحة هي 120 × 80 = 9600 متر مربع",
      "mark": "2"
    },
    {
      "type": "image",
      "src": "/uploads/land-shape.jpg",
      "caption": "شكل تقريبي لقطعة الأرض",
      "width": 300
    },
    {
      "type": "text",
      "content": "2) غرس الفلاح 1/3 المساحة أشجاراً. ما هي المساحة المغروسة؟",
      "answer": "9600 ÷ 3 = 3200 متر مربع",
      "mark": "3"
    }
  ]
}
```

## 3. قالب موضوع كامل (امتحان / فرض)

الموضوع الكامل يتكون من **معلومات عامة (Metadata)**، و **قائمة من التمارين (Exercises)** التي تم شرحها في الأعلى. عند إدخال موضوع امتحان رسمي للرياضيات 1 متوسط - الفصل الأول، يجب تنظيمه كالتالي:

```json
{
  "metadata": {
    "levelId": "moyen",
    "yearId": "1am",
    "yearName": "الأولى متوسط",
    "subjectId": "MAT",
    "subjectName": "الرياضيات",
    "trimesterId": "t1",
    "trimesterName": "الفصل الأول",
    "docType": "exam",
    "academicYear": "2024 / 2025",
    "duration": "ساعتان",
    "siteUrl": "SujetStore.com"
  },
  "exercises": [
    {
      "num": "التمرين الأول",
      "points": 3,
      "content": [
        { "type": "text", "content": "1) أنجز القسمة الإقليدية للعدد 145 على 6." }
      ]
    },
    {
      "num": "التمرين الثاني",
      "points": 4,
      "content": [
        { "type": "text", "content": "ارسم دائرة قطرها 5cm." }
      ]
    },
    {
      "num": "الوضعية الإدماجية",
      "points": 8,
      "content": [
        { "type": "text", "content": "لإقامة حفلة بالمدرسة، ساهم 5 تلاميذ بمبلغ..." }
      ]
    }
  ]
}
```

### تفاصيل الميتاداتا (`metadata`) لـ 1 متوسط رياضيات:
- `levelId`: يكون دائماً `moyen` للمتوسط.
- `yearId`: يكون `1am` للسنة الأولى متوسط.
- `trimesterId`: يرجى إدخال `t1` (للفصل 1)، `t2` (للفصل 2)، أو `t3` (للفصل 3).
- `docType`: يكون إما `exam` (اختبار) أو `test` (فرض).
- `duration`: يتم كتابة المدة كنص (مثال: "ساعتان"، "ساعة ونصف").

## 4. أنواع عناصر المحتوى المدعومة (Content Blocks)

داخل أي تمرين، نستخدم مصفوفة `content` لترتيب عناصر التمرين. هذه هي أهم الأنواع (types) المتاحة للإدخال:

### أ) النص (`text`)
لإضافة نصوص عادية، أسئلة، ومفردات. 
> [!NOTE]
> يمكن إضافة رموز رياضية **داخل النص** مباشرة (Inline Math) بوضعها بين علامتي دولار `$ $` باستخدام لغة **Typst**.
```json
{
  "type": "text",
  "content": "احسب قيمة المجهول $x$ إذا كان $x + 5 = 10$.",
  "answer": "$x = 5$",
  "mark": "1.5"
}
```

### ب) الرياضيات والمعادلات (`math`)
لكتابة معادلات رياضية كبيرة في سطر مستقل (استخدم صيغة **Typst** للمعادلات بدلاً من LaTeX).
```json
{
  "type": "math",
  "content": "A = 1/2 + 3.5",
  "display": true
}
```
*(ملاحظة: `display = true` تجعل المعادلة في سطر مستقل وحجم كبير).*

### ج) الصور والأشكال الهندسية (`image`)
لإدراج الصور الخارجية (مثل الصور الفوتوغرافية أو الرسوم المعقدة الجاهزة).
```json
{
  "type": "image",
  "src": "triangle_abc.png",
  "caption": "المثلث القائم ABC",
  "width": 200
}
```
> [!IMPORTANT]
> **بالنسبة للصور**: يكفي كتابة اسم الملف (مثلاً `triangle.png`). يجب تجميع كل الصور المستخدمة في مجلد واحد وإرسالها مع ملف الـ JSON.

### د) الرسوم الهندسية البرمجية (Typst Drawing)
يمكن للمتخصصين رسم الأشكال الهندسية (مثل المثلثات، الدوائر، الزوايا) مباشرة داخل النص باستخدام لغة **Typst** وحزمة **Cetz**. 
للقيام بذلك، نضع كود الرسم داخل حقل `content` من النوع `text`.

```json
{
  "type": "text",
  "content": "لاحظ الشكل التالي: #canvas({
    import draw: *
    line((0,0), (3,0), (1.5,2), close: true)
    content((1.5,-0.3), [3cm])
  })"
}
```
*(ملاحظة: هذا الخيار مخصص لمن يتقنون لغة الرسم في Typst، وهو يغني عن إرفاق صور خارجية للأشكال البسيطة).*

### هـ) صح أو خطأ (`true_false`)
لتمارين التقييم المباشر مع إمكانية إضافة تصحيح للخطأ.
```json
{
  "type": "true_false",
  "items": [
    {
      "q": "العدد 15 يقبل القسمة على 2.",
      "a": "خطأ",
      "c": "لأن آحاده ليس زوجياً.",
      "mark": "1"
    },
    {
      "q": "المثلث القائم له زاوية 90 درجة.",
      "a": "صحيح",
      "mark": "1"
    }
  ]
}
```

### و) اختيار من متعدد (`multiple_choice`)
الأسئلة التي تحتوي على عدة خيارات.
```json
{
  "type": "multiple_choice",
  "groups": [
    {
      "header": "ما هو ناتج 5 × 4 ؟",
      "options": ["10", "20", "25"],
      "correct": "20",
      "mark": "1"
    }
  ]
}
```

### ز) مخطط تدفقي / سلسلة عمليات (`diagram_flow`)
لتمثيل العمليات الحسابية المتسلسلة (مدخل -> عملية -> مخرج).
```json
{
  "type": "diagram_flow",
  "flow": ["10", "× 2", "20"],
  "mark": "1.5"
}
```

### ح) وضع تسميات / ملء فراغات (`labeling`)
لإدراج قائمة من الكلمات التي يجب وضعها في مكانها الصحيح.
```json
{
  "type": "labeling",
  "labels": ["البسط", "المقام", "خط الكسر"],
  "mark": "2"
}
```

### ط) شبكة صور مع تسميات (`image_grid`)
لمقارنة صور متعددة أو تسميتها في جدول منظم.
```json
{
  "type": "image_grid",
  "columns": 2,
  "items": [
    { "src": "angle1.png", "label": "زاوية حادة", "answer": "صحيح", "mark": "0.5" },
    { "src": "angle2.png", "label": "زاوية منفرجة", "answer": "صحيح", "mark": "0.5" }
  ]
}
```

### ي) الجداول (`table`)
للجداول مثل جداول التناسبية.
```json
{
  "type": "table",
  "headers": ["الكمية (kg)", "السعر (DA)"],
  "cells": [
    "2",
    { "content": "150", "answer": "150", "mark": "0.5" },
    "5",
    { "content": "?", "answer": "375", "mark": "1" }
  ]
}
```

---

> [!TIP]
> **نصيحة لمن يملأ البيانات**: 
> - تأكد دائماً من أن مجموع الـ `points` في التمارين المتفرقة يساوي 20 (النقطة الكاملة للموضوع).
> - المعادلات الرياضية في حقل `math` يجب أن تُكتب باستخدام قواعد محرك **Typst** (مثلا: الكسر يكتب `1/2` أو `frac(1, 2)` والضرب يكتب `times`، إلخ).
> - النظام يدعم مكتبة **Cetz** للرسم الهندسي البرمجي داخل النصوص.
